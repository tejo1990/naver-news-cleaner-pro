## 개요
코드 수정 후 브라우저에서 직접 검증하고, 결과를 문서로 남기는 표준 워크플로우.
유저가 콘솔 로그를 직접 복사해 전달하는 비효율을 없애고,
에이전트가 자율적으로 검증-재수정-문서화 루프를 완성한다.

---

## Step 1: 코드 수정
코드 변경 사항을 에디터에서 저장한다.

## Step 2: 확장 프로그램 강제 재로드
브라우저 서브에이전트로 chrome://extensions 에서 확장 프로그램을 새로고침한다.

```
browser_subagent:
  1. chrome://extensions 열기
  2. "네이버 뉴스 클리너" 카드에서 새로고침(🔄) 버튼 클릭
  3. 스크린샷으로 재로드 완료 확인
```

## Step 3: 네이버 뉴스 페이지에서 콘솔 로그 검증
```
browser_subagent:
  1. https://news.naver.com 열기
  2. 팝업에서 테스트용 언론사 추가 (예: "연합뉴스")
  3. 페이지 새로고침 (F5)
  4. 콘솔에서 다음 확인:
     - "[NNC]" 접두사 로그 존재 여부
     - "필터 준비 완료" 메시지
     - "차단됨" 또는 "통과" 로그
     - 오류 메시지 없음 확인
  5. 스크린샷 저장
```

## Step 4: 검증 판단 기준
- ✅ 성공: "[NNC] 스캔 완료" 로그 + 차단 기사 숨김 애니메이션 확인
- ❌ 실패: EvalError / TypeError / 차단 0개 (설정 있는데)

### 실패시 → Step 5로
### 성공시 → Step 6으로

## Step 5: 자동 재수정
실패 로그를 분석해 원인을 파악하고 코드를 수정한 후 Step 2로 돌아간다.
최대 3회 시도 후에도 실패 시 사용자에게 상황 보고.

## Step 6: 결과 문서 작성
`docs/debug_sessions/YYYY-MM-DD_[주제].md` 파일을 생성한다.

포함 내용:
- 날짜 및 세션 개요
- 문제 증상 (초기 상태)
- 근본 원인 분석
- 시도한 해결 방법들 (실패한 것 포함)
- 최종 해결 방법
- 검증 결과 (콘솔 로그, 스크린샷 경로)
- 유지보수 시 주의사항

## 사용법
이 워크플로우를 실행하려면:
```
/extension-debug-workflow
```
또는 대화에서 "워크플로우 실행해줘" 라고 말하면 된다.
